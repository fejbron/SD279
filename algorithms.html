Why Sorting & Searching?

  • Sorting is a fundamental building block — used in databases, compilers, OS schedulers
  • Searching a sorted list is orders of magnitude faster than unsorted
  • Most real-world data pipelines involve at least one sort
  • Sorting enables binary search, merging, deduplication, and ranking
  • Understanding trade-offs determines which algorithm to pick for a given problem

"Sorting is the most fundamental algorithmic problem in computer science."
— Donald Knuth

Did you know?

Google reportedly spends 20–25% of its total compute cycles on sorting operations.

Key Properties of Sorting Algorithms

Stable Sort

Equal elements maintain their original relative order. Critical when sorting by multiple keys (e.g., sort students by grade, then by name).

Unstable Sort

Equal elements may be reordered. Faster in practice but unsafe for multi-key sorting.

Adaptive Sort

Runs faster on partially sorted input. Insertion Sort is O(n) for nearly sorted data.

In-Place Sort

Uses O(1) extra memory — sorts within the original array. Examples: Bubble, Selection, Insertion, Heap Sort.

Not In-Place

Requires O(n) or more extra memory. Examples: Merge Sort needs an auxiliary array.

Knowing these properties helps you choose the right sort for the right constraint.

Bubble Sort

Repeatedly swap adjacent elements if they are in the wrong order. Large elements "bubble" to the end each pass.

def bubble_sort(arr):
  n = len(arr)
  for i in range(n):
    for j in range(0, n-i-1):
      if arr[j] > arr[j+1]:
        arr[j], arr[j+1] = arr[j+1], arr[j]
Optimisation

Add a swapped flag — stop early if no swaps occurred. Best-case becomes O(n) for sorted input.

Selection Sort

Find the minimum element in the unsorted portion and swap it to its correct position. Repeat for each position.

def selection_sort(arr):
  n = len(arr)
  for i in range(n):
    min_idx = i
    for j in range(i+1, n):
      if arr[j] < arr[min_idx]:
        min_idx = j
    arr[i],arr[min_idx]=arr[min_idx],arr[i]

Makes at most n-1 swaps — great when writes are expensive (e.g., flash memory).

Insertion Sort

Build the sorted array one element at a time by inserting each element into its correct position among the already-sorted elements.

def insertion_sort(arr):
  for i in range(1, len(arr)):
    key = arr[i]
    j = i - 1
    while j >= 0 and arr[j] > key:
      arr[j+1] = arr[j]
      j -= 1
    arr[j+1] = key
Best use case

Small arrays or nearly-sorted data. It's the algorithm used in practice for small subarrays inside Timsort and Introsort.

Merge Sort — Divide & Conquer

Recursively split the array in half until single elements, then merge sorted halves back together.

  • Divide: Split array into two halves — O(log n) levels
  • Conquer: Recursively sort each half
  • Merge: Combine two sorted halves — O(n) per level
  • Total: O(n) × O(log n) = O(n log n) guaranteed
Key advantage

Stable sort with guaranteed O(n log n) worst case. Ideal for linked lists and external sorting (data too large for RAM).

Merge Sort — Implementation

def merge_sort(arr):
  if len(arr) <= 1:
    return arr
  mid = len(arr) // 2
  left = merge_sort(arr[:mid])
  right = merge_sort(arr[mid:])
  return merge(left, right)

def merge(L, R):
  result, i, j = [], 0, 0
  while i < len(L) and j < len(R):
    if L[i] <= R[j]:
      result.append(L[i]); i += 1
    else:
      result.append(R[j]); j += 1
  result += L[i:] + R[j:]
  return result
  • Base case: array of length ≤ 1 is already sorted
  • Two recursive calls halve the problem size each time
  • The merge() function compares front elements and builds a sorted result
  • Remaining elements appended directly — no extra comparisons needed

The merge step is where all the work happens — always O(n) per level.

Real-world use

Python's built-in sorted() and Java's Arrays.sort() for objects both use a variant of Merge Sort (Timsort).

Quick Sort

Pick a pivot, partition the array so all elements less than the pivot are on the left, greater on the right. Recursively sort both sides.

  • Partition is the key step — rearranges in-place
  • Pivot choice matters: first, last, median, or random
  • Average O(n log n), worst case O(n²) on sorted input
  • In-place and cache-friendly — often faster than Merge Sort in practice

Quick Sort — Implementation

def quick_sort(arr, lo, hi):
  if lo < hi:
    p = partition(arr, lo, hi)
    quick_sort(arr, lo, p-1)
    quick_sort(arr, p+1, hi)

def partition(arr, lo, hi):
  pivot = arr[hi] # last elem
  i = lo - 1
  for j in range(lo, hi):
    if arr[j] <= pivot:
      i += 1
      arr[i],arr[j] = arr[j],arr[i]
  arr[i+1],arr[hi]=arr[hi],arr[i+1]
  return i + 1
  • lo and hi define the subarray range
  • Partition returns the final index of the pivot
  • All elements left of pivot < pivot; right > pivot
  • Recursion is tail-call on one side — space-optimisable
Avoid worst case

Use random pivot or median-of-three to avoid O(n²) on already-sorted input.

Introsort

C++ STL std::sort uses Introsort: Quick Sort + falls back to Heap Sort if recursion depth exceeds O(log n).

Heap Sort

Build a max-heap from the array, then repeatedly extract the maximum element and place it at the end.

  • Phase 1 – Heapify: Build max-heap in O(n)
  • Phase 2 – Extract: Swap root with last, reduce heap size, restore heap — O(log n) per extraction
  • Total: O(n log n) guaranteed — even worst case
  • In-place but not stable
def heap_sort(arr):
  n = len(arr)
  for i in range(n//2-1, -1, -1):
    heapify(arr, n, i) # build heap
  for i in range(n-1, 0, -1):
    arr[0], arr[i] = arr[i], arr[0]
    heapify(arr, i, 0) # restore
Max-Heap property

Parent ≥ children at every node. Root always holds the maximum element.

heapify(arr, n, i)

Ensures the subtree rooted at index i satisfies the heap property. Runs in O(log n).

Counting Sort

Count occurrences of each value, compute cumulative counts to determine positions, then place each element in its correct output position.

def counting_sort(arr, k):
  # k = max value in arr
  count = [0] * (k+1)
  for x in arr:
    count[x] += 1
  # cumulative counts
  for i in range(1, k+1):
    count[i] += count[i-1]
  output = [0] * len(arr)
  for x in reversed(arr):
    output[count[x]-1] = x
    count[x] -= 1
  return output
  • Does not compare elements — beats O(n log n) lower bound
  • Time: O(n + k) where k = range of values
  • Space: O(n + k) — needs auxiliary arrays
  • Only works for integer keys within a known range
  • Stable — iterating in reverse preserves order

Best for small range k (e.g., grades 0–100, ages 0–150). Wasteful if k ≫ n.

Radix Sort

Sort integers digit by digit, from least significant to most significant digit (LSD Radix Sort), using a stable sort on each digit pass.

  • Uses Counting Sort as the stable subroutine per digit
  • d passes (one per digit), each O(n + b) where b = base (usually 10)
  • Total time: O(d · (n + b)) — effectively O(n) for fixed-width integers
  • Stable, not in-place. Works for strings too (sort char by char)

Linear Search

Scan every element from left to right until the target is found or the list is exhausted. No pre-conditions required.

def linear_search(arr, target):
  for i, val in enumerate(arr):
    if val == target:
      return i # index found
  return -1 # not found

Binary Search

Requires a sorted array. Compare target with the middle element — eliminate half the search space each step.

def binary_search(arr, target):
  lo, hi = 0, len(arr) - 1
  while lo <= hi:
    mid = (lo + hi) // 2
    if arr[mid] == target:
      return mid
    elif arr[mid] < target:
      lo = mid + 1
    else:
      hi = mid - 1
  return -1

Binary Search — Variants & Applications

Find First Occurrence

When duplicates exist, binary search can be modified to find the leftmost occurrence by continuing to search left even after a match (hi = mid - 1).

Find Last Occurrence

Symmetric to above — continue searching right after a match (lo = mid + 1) to find the rightmost duplicate.

Search on Answer (Binary Search on Value)

Apply binary search on the answer space rather than an array — e.g. "find smallest x such that f(x) is true." Used in optimisation problems.

Rotated Sorted Array

Binary search still works in O(log n) on a rotated sorted array by determining which half is sorted at each step.

Recursive vs Iterative

Iterative is O(1) space. Recursive is O(log n) space due to call stack. Always prefer iterative unless clarity demands recursion.

Binary Search is one of the most frequently tested algorithms in technical interviews.

Choosing the Right Algorithm

Small n (< 50)

Insertion Sort. Low overhead, cache-friendly, O(n) on nearly sorted data. Used inside Timsort for small runs.

General purpose

Quick Sort (randomised pivot). Fastest average case in practice. Default in most standard libraries for primitives.

Need stability

Merge Sort or Timsort. Required when sorting objects by multiple fields or preserving original order of equal keys.

Memory constrained

Heap Sort. Guaranteed O(n log n) worst-case, O(1) space. Used when neither recursion nor extra arrays are affordable.

Integer keys, small range

Counting Sort. Outperforms all comparison-based sorts when k is small relative to n.

Large integers / strings

Radix Sort. Sorts fixed-width keys in linear time. Used in network packet routing and suffix array construction.

The Ω(n log n) Lower Bound

Any comparison-based sorting algorithm requires at least Ω(n log n) comparisons in the worst case.

  • To sort n elements, we must distinguish between n! possible orderings (permutations)
  • A comparison produces 1 bit of information — binary decision tree
  • Tree must have ≥ n! leaves, so height ≥ log₂(n!)
  • By Stirling's approximation: log₂(n!) ≈ n log₂ n − n log₂ e = Θ(n log n)
  • Therefore: Merge Sort, Heap Sort, and Quick Sort (avg) are asymptotically optimal
Implication

You cannot sort faster than O(n log n) using comparisons. To go faster you must exploit additional structure (e.g., integer keys → Counting/Radix Sort).

Decision tree for n=3

3! = 6 leaves required. Tree height ≥ ⌈log₂ 6⌉ = 3. Means at least 3 comparisons needed to sort 3 elements.

Real-World Implementations

Python — Timsort

Hybrid of Merge Sort + Insertion Sort. Finds natural runs in data, merges them efficiently. Worst case O(n log n), best O(n). Used by sorted() and .sort().

Java — Dual-Pivot Quick Sort

Arrays of primitives: Dual-Pivot Quick Sort (two pivots, 3-way partition). Objects: Timsort (stable required). Found in Arrays.sort().

C++ — Introsort

Quick Sort + Heap Sort fallback when depth exceeds 2·log(n). Guarantees O(n log n) worst case. Used in std::sort.

Databases — External Sort

Data too large for RAM: External Merge Sort. Sort chunks that fit in memory, then merge them from disk. Used in PostgreSQL, MySQL.

Parallel Sorting

Merge Sort parallelises naturally — split into threads. Bitonic Sort for GPU-parallel sorting. MapReduce uses external sort for large distributed datasets.

No single algorithm wins everywhere. The best implementations combine multiple strategies.

Key Takeaways

  • Big-O describes growth rate — always report worst case unless stated
  • O(n²) sorts (Bubble, Selection, Insertion) — simple, good for small or nearly-sorted data
  • O(n log n) sorts (Merge, Quick, Heap) — optimal for general comparison-based sorting
  • Merge Sort: stable, guaranteed O(n log n), uses O(n) extra space
  • Quick Sort: fastest in practice, O(n²) worst case — randomise the pivot
  • Heap Sort: O(n log n) guaranteed, in-place, not stable
  • Counting & Radix Sort beat O(n log n) by not comparing elements
  • Linear Search: O(n), works on any list
  • Binary Search: O(log n), requires sorted array — up to 10⁹× faster
  • No sorting algorithm is universally best — constraints determine the choice
