What Is a Tree?

  • A hierarchical non-linear data structure
  • Consists of nodes connected by edges
  • Has a single root node at the top
  • Each node has at most one parent
  • Nodes with no children are called leaves
  • Unlike arrays/lists — not sequential

Key Terminology

Root

The topmost node — has no parent

Longest path from root to a leaf node

Depth

Distance from the root to a given node

Number of children a node has

Subtree

Any node plus all of its descendants

Binary Trees

Each node has at most 2 children: a left child and a right child.

  • Full BT — every node has 0 or 2 children
  • Complete BT — all levels filled except possibly the last
  • Perfect BT — all internal nodes have 2 children, all leaves same depth
  • Degenerate (Skewed) — every node has only one child

Node Structure in Code

A node stores a value and references to its left and right children.

class Node:
  def __init__(self, val):
    self.val = val
    self.left = None
    self.right = None

The tree itself can just be a reference to the root node.

# Build a simple tree
root = Node(1)
root.left = Node(2)
root.right = Node(3)
root.left.left = Node(4)
root.left.right = Node(5)

Depth-First Traversals

Inorder (L → Root → R)

Visits left subtree, then root, then right. Produces sorted output on a BST.

def inorder(node):
  if node:
    inorder(node.left)
    print(node.val)
    inorder(node.right)
Preorder (Root → L → R)

Visits root first — useful for copying or serializing a tree.

Postorder (L → R → Root)

Visits root last — useful for deleting a tree or evaluating expressions.

def postorder(node):
  if node:
    postorder(node.left)
    postorder(node.right)
    print(node.val)

All three are O(n) — every node visited exactly once.

08 / 18

Level-Order Traversal (BFS)

  • Visits nodes level by level, left to right
  • Uses a queue (FIFO) — not recursion
  • Useful for finding shortest path, printing level by level
  • Time: O(n) — Space: O(w) where w = max width
from collections import deque
def bfs(root):
  q = deque([root])
  while q:
    node = q.popleft()
    print(node.val)
    if node.left: q.append(node.left)
    if node.right: q.append(node.right)

The BST Property

For every node N:
All values in left subtree < N.val
All values in right subtree > N.val

  • This holds recursively for every subtree
  • No duplicate values in a standard BST
  • Inorder traversal of a BST → sorted sequence

BST — Search

  • Start at the root
  • If target = current → found!
  • If target < current → go left
  • If target > current → go right
  • Repeat until found or null
def search(node, target):
  if not node or node.val == target:
    return node
  if target < node.val:
    return search(node.left, target)
  return search(node.right, target)

BST — Insert

  • Follow search path to find the correct position
  • New node always inserted as a leaf
  • BST property must remain intact after insertion
  • Time complexity: O(h) where h = height
def insert(node, val):
  if not node:
    return Node(val)
  if val < node.val:
    node.left = insert(node.left, val)
  elif val > node.val:
    node.right = insert(node.right, val)
  return node

BST — Delete (3 Cases)

Case 1

Leaf Node

Node has no children. Simply remove it — no restructuring needed.

Case 2

One Child

Replace the node with its only child. Bypass the deleted node.

Case 3 ★

Two Children

Replace value with inorder successor (smallest in right subtree), then delete that node.

Delete is the trickiest BST operation — always check which case applies first.

Time Complexity

Operation Average Worst
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)
TraversalO(n)O(n)
SpaceO(n)O(n)

Worst case O(n) happens when the tree is skewed — essentially a linked list.

Why O(log n)?

A balanced tree of height h has ~2h nodes. So h ≈ log₂(n). Each operation traverses at most h levels.

Self-Balancing Trees & Applications

Self-Balancing BSTs

  • AVL Tree — Strictly balanced, height diff ≤ 1 per node
  • Red-Black Tree — Loosely balanced, used in Java's TreeMap
  • B-Tree — Multi-child, used in databases & file systems

These guarantee O(log n) in the worst case by re-balancing after insert/delete.

Real-World Applications

  • Database indexing (B+ Trees)
  • Autocomplete & symbol tables in compilers
  • Priority queues (Heap = special tree)
  • Expression parsing in calculators
  • File system directory structures
  • Network routing tables

Key Takeaways

  • A Binary Tree has at most 2 children per node
  • Traversals: Inorder, Preorder, Postorder (DFS) and Level-order (BFS)
  • A BST enforces left < root < right at every node
  • BST Inorder traversal gives a sorted sequence
  • All BST ops are O(h) — height matters!
  • Balanced tree → O(log n) · Skewed tree → O(n)
  • Use AVL or Red-Black trees to guarantee balance
  • Delete has 3 cases — always handle 2-child case carefully
