What Is a Data Structure?

A data structure is a way of organising, storing, and managing data so it can be accessed and modified efficiently.

  • A container that holds data in a particular format
  • Defines the relationships between data elements
  • Defines the operations that can be applied to the data
  • The choice of data structure directly affects algorithm performance
  • Wrong choice → slow, memory-hungry, bug-prone programs
Primitive DS

int, float, char, boolean — directly operated on by the CPU

Linear DS

Arrays, Linked Lists, Stacks, Queues — sequential arrangement

Non-Linear DS

Trees, Graphs, Heaps — hierarchical or networked arrangement

Hash-based DS

Hash Maps, Hash Sets — key-value with O(1) access

What Is an Algorithm?

An algorithm is a finite, step-by-step, unambiguous set of instructions for solving a problem or accomplishing a task.

  • Input: Zero or more well-defined inputs
  • Output: At least one well-defined output
  • Definiteness: Every step is clear and unambiguous
  • Finiteness: Terminates after a finite number of steps
  • Effectiveness: Each step is basic and executable

Data Structure + Algorithm = Program

— Niklaus Wirth

Correctness

Produces the right output for all valid inputs, including edge cases

Efficiency

Uses the minimum time and space necessary to solve the problem

Simplicity

Easy to understand, implement, and maintain

Abstract Data Types (ADTs)

An ADT defines what a data structure does (its interface) without specifying how it does it (its implementation).

  • Separates logical behaviour from implementation details
  • A Stack ADT says: push, pop, peek — but not whether it uses an array or linked list
  • Enables multiple implementations with the same interface
  • Foundation of object-oriented design and interfaces
ADTOperationsImplemented by
Listget, set, insert, removeArray, Linked List
Stackpush, pop, peekArray, Linked List
Queueenqueue, dequeueArray, Linked List
Mapput, get, deleteHash Table, BST
Setadd, remove, containsHash Set, BST Set

"Users of an ADT need not know its implementation — only its contract."

Complexity Analysis — Quick Recap

Big-O notation describes how an algorithm's runtime or space grows as input size n increases — ignoring constants and lower-order terms.

NotationGrowthExample
O(1)ConstantArray index
O(log n)LogarithmicBinary search
O(n)LinearLinear scan
O(n log n)LinearithmicMerge sort
O(n²)QuadraticBubble sort
Time Complexity

How many operations does the algorithm perform as n grows? Always report worst-case unless stated.

Space Complexity

How much extra memory does it use? O(1) = in-place. O(n) = extra array needed.

Best / Avg / Worst

Same algorithm can have different complexities depending on the input. Always reason about all three.

O(1) < O(log n) < O(n) < O(n log n) < O(n²)

What Is an Array?

An array is a collection of elements of the same data type, stored in contiguous (adjacent) memory locations, accessible by an integer index.

  • Stores a fixed-size sequence of elements (static array)
  • Index starts at 0 in most languages (C, Java, Python)
  • All elements occupy the same amount of memory
  • Random access in O(1) — direct address computation
  • Declared with a type and size: int arr[5]

Memory Layout of Arrays

Arrays store elements in a contiguous block of memory. The CPU can compute any element's address instantly — no traversal needed.

# int array: 4 bytes each
arr = [10, 20, 30, 40]
# base address = 1000
# arr[0] → 1000
# arr[1] → 1000 + 1×4 = 1004
# arr[2] → 1000 + 2×4 = 1008
# arr[3] → 1000 + 3×4 = 1012
  • O(1) access because address is a simple arithmetic formula
  • Cache-friendly — CPU loads a whole cache line of adjacent elements at once
  • This makes arrays the fastest data structure for sequential access

Array Operations — Access & Search

Access — O(1)

Retrieve any element directly by index. Fastest possible operation — single memory read.

arr = [5, 12, 8, 3, 17]
x = arr[2] # x = 8, O(1)
arr[0] = 99 # update in-place, O(1)

Linear Search — O(n)

Scan every element until target found. Works on any array, sorted or not.

def linear_search(arr, target):
  for i, val in enumerate(arr):
    if val == target: return i
  return -1

Binary Search — O(log n)

Requires a sorted array. Halves the search space every step — extremely fast for large arrays.

def binary_search(arr, target):
  lo, hi = 0, len(arr)-1
  while lo <= hi:
    mid = (lo+hi)//2
    if arr[mid]==target: return mid
    elif arr[mid]<target: lo=mid+1
    else: hi=mid-1
  return -1

Binary search on 1 billion elements: max 30 comparisons.

Static vs. Dynamic Arrays

Static Array

  • Fixed size declared at compile time
  • Cannot grow or shrink at runtime
  • Memory allocated on the stack (fast) or heap
  • Used in C: int arr[10];
  • Waste space if not full; overflow if too small

Dynamic Array

  • Starts small, resizes automatically when full
  • Doubles capacity when capacity exceeded (amortised O(1) append)
  • Memory allocated on the heap
  • Python list, Java ArrayList, C++ vector
  • Resize copies all elements — O(n), but rare enough to amortise to O(1)
Amortised Analysis

n appends at most trigger O(log n) resizes. Total copies: n + n/2 + n/4 + … ≤ 2n = O(n). So each append is O(1) amortised.

Common Array Patterns

Two Pointers

Use two indices (lo and hi) moving toward each other. Used for: pair sum, palindrome check, sorted array manipulation. O(n) instead of O(n²).

Sliding Window

Maintain a window of fixed or variable size, slide it across the array. Used for: max sum subarray, longest substring. O(n).

Prefix Sum

Precompute cumulative sums so range queries cost O(1) instead of O(n). prefix[i] = arr[0]+…+arr[i]

# Two Pointers — check palindrome
def is_palindrome(arr):
  lo, hi = 0, len(arr)-1
  while lo < hi:
    if arr[lo]!=arr[hi]: return False
    lo+=1; hi-=1
  return True

# Sliding Window — max sum of size k
def max_sum(arr, k):
  window = sum(arr[:k])
  best = window
  for i in range(k, len(arr)):
    window += arr[i] - arr[i-k]
    best = max(best, window)
  return best

# Prefix Sum build
prefix = [0] * (len(arr)+1)
for i, v in enumerate(arr):
  prefix[i+1] = prefix[i] + v

Common Array Mistakes & Edge Cases

Off-by-One Error

Valid indices are 0 to n-1. Accessing index n causes an IndexError. Use range(len(arr)), not range(len(arr)+1).

Empty Array

Always check if not arr or len(arr) == 0 before accessing elements or doing binary search.

Mutating While Iterating

Never remove elements from a list while looping over it — iterate over a copy or build a new list.

Shallow Copy

b = a does NOT copy a list — both point to the same object. Use b = a[:] or b = a.copy().

Integer Overflow

In languages like C/Java, mid = (lo+hi)/2 can overflow. Use mid = lo + (hi-lo)/2 for safety.

Single Element

Many algorithms break on arrays of size 1. Test this explicitly: if len(arr) == 1: return arr[0].

Negative Indices (Python)

Python allows negative indices: arr[-1] is the last element. Convenient but can hide bugs in other languages.

Wrong Slice

arr[1:3] includes indices 1 and 2 — not 3. The end index is exclusive in Python.

Classic Array Problem — Maximum Subarray (Kadane's)

Given an integer array, find the contiguous subarray with the largest sum.

  • Brute force: Try all subarrays — O(n²) or O(n³)
  • Kadane's algorithm: O(n) — greedily extend or restart
  • Key insight: if running sum goes negative, start fresh from current element
Trace: [-2, 1, -3, 4, -1, 2, 1, -5, 4]

Max subarray = [4, -1, 2, 1] → sum = 6

# Kadane's Algorithm — O(n)
def max_subarray(arr):
  best = arr[0]
  curr = arr[0]
  
  for n in arr[1:]:
    # extend current or restart
    curr = max(n, curr + n)
    best = max(best, curr)
  
  return best

arr = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
print(max_subarray(arr)) # 6

# O(n) time · O(1) space
# Dynamic programming pattern!

Applications

Real-World Applications of Arrays

Images

A bitmap image is a 2D array of pixels. RGB values are stored as integer triplets.

Audio

Audio samples stored as 1D array of amplitude values. MP3/WAV files are sequences of numbers.

Databases

Columnar storage: each column is an array. Enables fast scans for aggregation queries.

Machine Learning

NumPy arrays (tensors) power all ML frameworks. Matrix multiplication is the core operation.

Stacks & Queues

Both can be implemented using arrays. Stack: use end as top. Queue: circular buffer.

Heaps

A binary heap is stored as an array. Parent of node i is at (i-1)/2; children at 2i+1, 2i+2.

Hash Tables

The underlying structure of a hash map is an array of buckets indexed by hash value.

String Processing

Strings are character arrays. Substring search, pattern matching, and parsing all use array ops.

Summary

When to Use Arrays

Use an Array when…

  • You need fast random access by index
  • You know the size in advance (or use a dynamic array)
  • You're doing sequential processing — cache locality matters
  • You need the underlying structure for stacks, heaps, hash tables
  • You're working with mathematical data (matrices, vectors)

Avoid Arrays when…

  • You frequently insert or delete in the middle → use Linked List
  • You need key-value lookup → use Hash Map
  • You need sorted dynamic data → use BST or sorted list
  • You need priority ordering → use Heap
  • You need graph relationships → use adjacency list/matrix

The best engineers don't memorise rules — they understand why each structure exists.
