Introduction

What Is a Linked List?

A linked list stores nodes in non-contiguous memory. Each node holds data and a pointer to the next node.

  • No fixed size — grows & shrinks dynamically
  • No random access — must traverse from head
  • Efficient front insert/delete — O(1)
  • Extra memory per node for the pointer field
  • Foundation for stacks, queues, LRU cache

Traversal — O(n)

Start at head, follow next pointers until None. Every element visited once.

def traverse(self):
  curr = self.head
  while curr is not None:
    print(curr.data)
    curr = curr.next

def search(self, target):
  curr = self.head
  while curr:
    if curr.data == target:
      return curr
    curr = curr.next
  return None

Insert at Head — O(1)

Create new node, set its next to current head, update head. Two pointer assignments — always O(1).

def insert_head(self, data):
  node = Node(data)
  node.next = self.head
  self.head = node
  self.size += 1
Why O(1)?

No traversal — we always know the head. Two pointer updates regardless of list length.

Insert at Tail & at Position

Insert at Tail — O(n) or O(1)*

def insert_tail(self, data):
  node = Node(data)
  if not self.head:
    self.head = node; return
  curr = self.head
  while curr.next:
    curr = curr.next
  curr.next = node
# *O(1) with tail pointer

Insert at Position k — O(k)

def insert_at(self, k, data):
  if k == 0:
    return self.insert_head(data)
  curr = self.head
  for _ in range(k-1):
    if not curr.next: break
    curr = curr.next
  node = Node(data)
  node.next = curr.next
  curr.next = node

Always set new_node.next first, then prev.next = new_node. Reversing this loses the rest of the list!

Delete a Node — O(n)

def delete(self, target):
  if not self.head: return
  if self.head.data == target:
    self.head = self.head.next
    self.size -= 1; return
  curr = self.head
  while curr.next:
    if curr.next.data == target:
      curr.next = curr.next.next
      self.size -= 1; return
    curr = curr.next
  • Find node before target — O(n)
  • Set prev.next = target.next — O(1)
  • Delete head is O(1) special case

Doubly Linked List

Each node has prev and next pointers. Allows O(1) deletion given a node pointer — no need to find the predecessor.

class DNode:
  def __init__(self, data):
    self.data = data
    self.prev = None  # ←
    self.next = None  # →

# O(1) delete given node:
node.prev.next = node.next
node.next.prev = node.prev

  • Traverse both directions
  • 2× pointer memory vs singly
  • Used in: LRU cache, deque, OS scheduling

Circular Linked List

Circular Linked List

The last node's next points back to head — forming a circle. No null terminator.

  • Can be singly or doubly circular
  • Traversal loops — must track start node
  • Can start from any node
  • Used in: round-robin scheduling, playlists, game turns
curr = head
while True:
  process(curr)
  curr = curr.next
  if curr == head: break

Reverse a Linked List — Iterative O(n) O(1)

Use three pointers — prev, curr, next — to flip each node's pointer in a single pass.

def reverse(self):
  prev = None
  curr = self.head
  while curr:
    nxt = curr.next   # save
    curr.next = prev  # flip
    prev = curr      # advance
    curr = nxt       # advance
  self.head = prev   # new head

Reverse — Recursive

def reverse_rec(self, node):
  if not node or not node.next:
    return node
  new_head = self.reverse_rec(node.next)
  node.next.next = node
  node.next = None
  return new_head

Recurse to the last node, then flip each pointer on the way back up. Space: O(n) call stack.

Cycle Detection — Floyd's Algorithm O(n) O(1)

Slow moves 1 step, fast moves 2. If a cycle exists they must meet. If fast hits null — no cycle. Also called "Tortoise and Hare".

def has_cycle(head):
  slow = fast = head
  while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast:
      return True
  return False
  • Time: O(n) · Space: O(1)
  • Find cycle start: reset one pointer to head, advance both by 1 — they meet at the entry point

Find Middle Node — O(n) O(1)

Slow moves 1 step, fast moves 2. When fast reaches the end, slow is at the middle.

def find_middle(head):
  slow = fast = head
  while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
  return slow
# [1,2,3,4,5] → 3 [1,2,3,4] → 3
Applications

Split for merge sort, palindrome check (reverse second half), find median of a stream.

Merge Two Sorted Lists — O(n+m)

Compare front nodes; append smaller; advance that pointer. Use a dummy head to simplify edge cases.

def merge_sorted(l1, l2):
  dummy = Node(0)
  curr = dummy
  while l1 and l2:
    if l1.data <= l2.data:
      curr.next=l1; l1=l1.next
    else:
      curr.next=l2; l2=l2.next
    curr=curr.next
  curr.next = l1 or l2
  return dummy.next

Classic Algorithms

Remove N-th Node From End — O(n) O(1)

Maintain a gap of n nodes. When fast hits null, slow is just before the target node.

def remove_nth(head, n):
  dummy = Node(0)
  dummy.next = head
  slow = fast = dummy
  for _ in range(n+1):
    fast = fast.next
  while fast:
    slow=slow.next;fast=fast.next
  slow.next = slow.next.next
  return dummy.next
Remove 2nd from end of 1→2→3→4→5

fast jumps 3 positions. Both advance until fast=null. slow.next = node 4 → remove it. Result: 1→2→3→5

Two-pointer gap pattern

Fixed gap between fast and slow solves many LL problems in a single O(n) pass — including intersection detection.

Dummy node simplifies the edge case where we delete the head node (n = length).

Palindrome Linked List — O(n) O(1)

Find middle → reverse second half → compare both halves. O(n) time, O(1) auxiliary space.

def is_palindrome(head):
  slow=fast=head
  while fast and fast.next:
    slow=slow.next;fast=fast.next.next
  prev,curr=None,slow
  while curr:
    nxt=curr.next;curr.next=prev
    prev=curr;curr=nxt
  l,r=head,prev
  while r:
    if l.data!=r.data: return False
    l=l.next;r=r.next
  return True

Comparison

Choosing the Right Structure

Array
  • Random access by index
  • Cache-friendly sequential processing
  • Binary search needed
  • Mostly reads, few inserts
Singly Linked List
  • Frequent front insert/delete
  • Memory-constrained (1 ptr/node)
  • Implementing stacks or queues
  • Size changes frequently
Doubly Linked List
  • O(1) delete with node pointer
  • Bidirectional traversal
  • LRU cache, deque, browser history
  • Memory OK for 2 ptrs/node

Python collections.deque is a doubly linked list — use it for production code.

Applications

Real-World Applications

Hash Table Chaining

Each bucket is a linked list of entries with the same hash. Used in Python dicts (pre-3.6), Java HashMap.

LRU Cache

Doubly linked list + hash map. Accessed nodes move to front in O(1). Least used is at the tail.

OS Kernel

Linux uses intrusive doubly linked lists (list_head) for processes, file descriptors, network interfaces.

Undo / Redo

Each state is a node. Undo pops the head. Redo pushes to a separate list.

Memory Allocator

Free lists track available blocks. O(1) insert/delete without contiguous storage.

Graph Adjacency List

Each vertex's neighbours stored as a linked list. Standard for sparse graphs — O(V+E) space.

Common Mistakes

Common Linked List Mistakes

Losing the list on insert

Setting prev.next = new_node before new_node.next = prev.next loses the rest. Always set new_node.next first.

Not checking empty list

Always check if not self.head before any operation to avoid AttributeError.

Off-by-one in kth node

Is k 0-indexed or 1-indexed? Walk k-1 steps for 1-indexed, k steps for 0-indexed. Test on a small example.

Forgetting to update tail

When maintaining a tail pointer, always update it when adding/removing the last node.

Infinite loop on cyclic list

Without Floyd's, traversing a cyclic list loops forever. Use fast/slow or a visited set.

Wrong stop condition

while curr: visits all nodes. while curr.next: stops before last. Mixing them causes bugs.

Complete Singly LL in Python

class LinkedList:
  def __init__(self):
    self.head=None;self.size=0

  def prepend(self, d):
    n=Node(d);n.next=self.head
    self.head=n;self.size+=1

  def append(self, d):
    n=Node(d)
    if not self.head:
      self.head=n;self.size+=1;return
    c=self.head
    while c.next: c=c.next
    c.next=n;self.size+=1

  def delete(self, d):
    if not self.head: return
    if self.head.data==d:
      self.head=self.head.next
      self.size-=1;return
    c=self.head
    while c and c.next:
      if c.next.data==d:
        c.next=c.next.next
        self.size-=1;return
      c=c.next

  def __repr__(self):
    v=[];c=self.head
    while c:v.append(str(c.data));c=c.next
    return " → ".join(v)+" → null"

Key Takeaways

  • Linked list nodes store data + a pointer in non-contiguous memory — dynamic, no pre-allocation
  • No random access — index i requires O(n) traversal from head
  • Insert/delete at head = O(1); anywhere else = O(n) to find the position
  • Always set new_node.next first before updating the previous pointer
  • Doubly LL: O(1) delete with node pointer; costs one extra pointer per node
  • Circular LL: last node → head; used in round-robin scheduling
  • Two-pointer (fast/slow): find middle, cycle detection, nth from end — O(n) O(1)
  • Floyd's algorithm detects cycles in O(n) without extra space
  • Reverse iteratively in O(n) O(1). Recursive is O(n) O(n) — prefer iterative
  • Real uses: LRU cache, hash chaining, OS kernel, collections.deque
