Stacks

What Is a Stack?

A stack is a linear data structure that follows the LIFO (Last In, First Out) principle — the last element added is the first one removed.

  • Think of a stack of plates — you add and remove from the top
  • Only the top element is accessible at any time
  • Two primary operations: push (add) and pop (remove)
  • Also supports peek (view top without removing)
  • All operations are O(1) — constant time

Stack Operations

push(item) — O(1)

Add an item to the top of the stack. If the stack is full (array-based), throw a Stack Overflow error.

pop() — O(1)

Remove and return the top item. If the stack is empty, throw a Stack Underflow error.

peek() / top() — O(1)

Return the top item without removing it. Useful for checking what's next without modifying the stack.

is_empty() — O(1)

Check if the stack has no elements. Always check before calling pop() to avoid underflow.

Stack — Array Implementation

class Stack:
  def __init__(self, cap=100):
    self.data = [None] * cap
    self.top = -1
    self.cap = cap

  def push(self, item):
    if self.top == self.cap - 1:
      raise OverflowError("Full")
    self.top += 1
    self.data[self.top] = item

  def pop(self):
    if self.is_empty():
      raise IndexError("Empty")
    val = self.data[self.top]
    self.top -= 1
    return val

  def peek(self):
    return self.data[self.top]

  def is_empty(self):
    return self.top == -1
  • Uses a fixed-size array; top tracks the last valid index
  • All operations O(1) — no traversal needed
  • Limitation: fixed capacity — must pre-allocate
  • In Python, use a plain list with append() and pop()

Stack — Linked List Implementation

class Node:
  def __init__(self, val):
    self.val = val
    self.next = None

class Stack:
  def __init__(self):
    self.head = None
    self.size = 0

  def push(self, val):
    node = Node(val)
    node.next = self.head # prepend
    self.head = node
    self.size += 1

  def pop(self):
    if not self.head: raise IndexError()
    val = self.head.val
    self.head = self.head.next
    self.size -= 1
    return val

  def peek(self):
    return self.head.val
  • No fixed size — grows dynamically, no overflow
  • Push = prepend to head (O(1))
  • Pop = remove head (O(1))
  • Extra memory per node for the next pointer

The head of the linked list is the top of the stack — all operations happen at the head in O(1).

Application: Balanced Brackets

Use a stack to check if every opening bracket has a matching closing bracket in the correct order.

def is_balanced(s):
  stack = []
  pairs = {')':'(', ']':'[', '}':'{'}
  for ch in s:
    if ch in '([{':
      stack.append(ch) # push
    elif ch in pairs:
      if not stack or stack[-1]!=pairs[ch]:
        return False
      stack.pop()
  return not stack # empty = balanced

Stacks — Applications

Stack Applications

Function Call Stack

Every function call pushes a stack frame. On return, the frame is popped. This is literally the CPU's call stack — stack overflow = too many nested calls.

Undo / Redo

Text editors push each action onto an undo stack. Ctrl+Z pops the last action. Ctrl+Y pushes it onto a redo stack. Used in Photoshop, VS Code, Word.

Browser History

Back/Forward navigation uses two stacks. Navigating pushes to the back stack. Clicking Back pops it and pushes to the forward stack.

Expression Evaluation

Convert infix to postfix (Shunting-Yard algorithm). Evaluate postfix expressions using a stack. Used in compilers and calculators.

DFS Graph Traversal

Depth-First Search uses a stack (explicitly or via recursion) to explore as deep as possible before backtracking. Used in maze solving and cycle detection.

Balanced Brackets / XML

Parsing nested structures: HTML tags, JSON brackets, code blocks. Any time you need to match open-close pairs.

Stacks — Patterns

Monotonic Stack Pattern

A monotonic stack maintains elements in increasing or decreasing order — pop elements that violate the order before pushing.

  • Used to find next greater/smaller element in O(n)
  • Used in: Largest Rectangle in Histogram, Daily Temperatures, Stock Span
  • Each element is pushed and popped at most once → O(n) total
# Next Greater Element — O(n)
def next_greater(arr):
  n = len(arr)
  result = [-1] * n
  stack = [] # stores indices

  for i in range(n):
    # pop while current > stack top
    while stack and arr[i]>arr[stack[-1]]:
      idx = stack.pop()
      result[idx] = arr[i]
    stack.append(i)

  return result

# [2,1,2,4,3] → [4,2,4,-1,-1]

What Is a Queue?

A queue is a linear data structure that follows the FIFO (First In, First Out) principle — the first element added is the first one removed.

  • Think of a queue at a shop — first person in line is served first
  • Elements enter at the rear (back)
  • Elements leave from the front
  • enqueue (add to back), dequeue (remove from front)
  • All operations are O(1)

Queue — Circular Buffer (Array)

A naive array queue wastes space as elements are dequeued. A circular buffer reuses slots by wrapping the front/rear indices around using modulo.

class Queue:
  def __init__(self, cap):
    self.data = [None] * cap
    self.front = 0
    self.size = 0
    self.cap = cap

  def enqueue(self, item):
    if self.size == self.cap: raise OverflowError()
    rear = (self.front + self.size) % self.cap
    self.data[rear] = item
    self.size += 1

  def dequeue(self):
    if self.size == 0: raise IndexError()
    val = self.data[self.front]
    self.front = (self.front+1) % self.cap
    self.size -= 1
    return val

The rear = (front + size) % cap formula wraps around — slots freed by dequeue are reused.

Queue — Linked List & Python

class Queue:
  def __init__(self):
    self.head = self.tail = None
    self.size = 0

  def enqueue(self, val):
    node = Node(val)
    if self.tail:
      self.tail.next = node
    self.tail = node
    if not self.head:
      self.head = node
    self.size += 1

  def dequeue(self):
    if not self.head: raise IndexError()
    val = self.head.val
    self.head = self.head.next
    if not self.head: self.tail = None
    self.size -= 1
    return val
  • Head = front (dequeue from here)
  • Tail = rear (enqueue here)
  • Both operations O(1) — tail pointer avoids traversal
  • No size limit — grows dynamically
Python — collections.deque
from collections import deque
q = deque()
q.append(10) # enqueue O(1)
q.append(20)
q.popleft() # dequeue → 10 O(1)
q[0] # peek front

Queues — Applications

Application: BFS with a Queue

Breadth-First Search uses a queue to visit all neighbours at distance k before moving to distance k+1 — guaranteeing shortest path in unweighted graphs/trees.

from collections import deque

def bfs(graph, start):
  visited = {start}
  q = deque([start])
  while q:
    node = q.popleft() # FIFO!
    print(node)
    for nb in graph[node]:
      if nb not in visited:
        visited.add(nb)
        q.append(nb)

Queue ensures level-by-level processing — all nodes at depth 1 before depth 2, etc.

Queue: [A]

Step 2 — dequeue A, enqueue B, C

Queue: [B, C]

Step 3 — dequeue B, enqueue D, E

Queue: [C, D, E]

FIFO order = level-order traversal

Visit: A → B → C → D → E → F

Queues — Applications

Queue Applications

CPU Scheduling

OS task scheduler uses a queue of processes. Round-robin scheduling: each process gets a time slice, then goes to the back of the queue.

Print Spooling

Print jobs queued FIFO. First document submitted is first printed. The OS manages the printer queue.

BFS / Level-Order

Graph and tree traversal. Used in: shortest path (unweighted), social network distance, web crawlers, peer discovery.

Network Buffers

Routers buffer packets in a queue. Network interface cards process outgoing packets in FIFO order to avoid collisions.

Message Queues

Kafka, RabbitMQ, SQS — distributed queues for async communication between services. Producer enqueues, consumer dequeues.

Keyboard Input

Keystrokes stored in a FIFO queue. The OS processes them in order to ensure characters appear correctly even when CPU is busy.

Queues — Variants

Deque — Double-Ended Queue

A deque supports insert and delete at both front and back in O(1) — generalising both stacks and queues.

  • addFront / addBack — O(1)
  • removeFront / removeBack — O(1)
  • Can simulate both Stack (addBack/removeBack) and Queue (addBack/removeFront)
  • Python collections.deque is a doubly-linked list — O(1) at both ends
  • Used in: sliding window maximum, palindrome checking, work stealing schedulers
from collections import deque
d = deque([2, 3, 4])

d.appendleft(1) # [1,2,3,4]
d.append(5) # [1,2,3,4,5]
d.popleft() # → 1, [2,3,4,5]
d.pop() # → 5, [2,3,4]
d[0] # peek front → 2
d[-1] # peek back → 4
Sliding Window Max — O(n)

Use a monotonic deque to track the maximum in a sliding window. Each element enters/exits at most once → O(n) total.

Queues — Variants

Priority Queue

A priority queue dequeues elements in order of their priority — not insertion order. Highest (or lowest) priority element is always served first.

  • Implemented with a Binary Heap — O(log n) insert and extract-min/max
  • Python: heapq module (min-heap by default)
  • Java: PriorityQueue<T>
  • Used in: Dijkstra's algorithm, A* search, task scheduling, Huffman coding
import heapq

pq = []
heapq.heappush(pq, (3, "task C"))
heapq.heappush(pq, (1, "task A"))
heapq.heappush(pq, (2, "task B"))

# Always pops min priority first
heapq.heappop(pq) # (1, "task A")
heapq.heappop(pq) # (2, "task B")
heapq.heappop(pq) # (3, "task C")

# Max-heap: negate priority
heapq.heappush(pq, (-10, "urgent"))

Classic Problems

Classic: Implement Queue Using Two Stacks

Use two stacks — inbox (push here) and outbox (pop from here) — to simulate FIFO behaviour with LIFO stacks.

class MyQueue:
  def __init__(self):
    self.inbox = []
    self.outbox = []

  def enqueue(self, val):
    self.inbox.append(val) # O(1)

  def dequeue(self):
    if not self.outbox:
      # pour inbox → outbox (O(n))
      while self.inbox:
        self.outbox.append(
          self.inbox.pop())
    return self.outbox.pop()

Each element is moved at most once from inbox to outbox → amortised O(1) per dequeue.

Enqueue 1, 2, 3

First dequeue — outbox empty

Pour inbox into outbox:
inbox: [] · outbox: [3, 2, 1]

Pop outbox → returns 1 ✓ FIFO!

outbox: [3, 2]

Second dequeue — outbox has elements

Pop outbox directly → returns 2 ✓

Classic: Implement Stack Using Queues

Use a single queue — on each push, enqueue the new element then rotate all existing elements behind it to make it the front.

from collections import deque

class MyStack:
  def __init__(self):
    self.q = deque()

  def push(self, val): # O(n)
    self.q.append(val)
    # rotate all others to back
    for _ in range(len(self.q)-1):
      self.q.append(self.q.popleft())

  def pop(self): # O(1)
    return self.q.popleft()

  def top(self):
    return self.q[0]

After each push, the newest element is rotated to the front — so it's always dequeued first (LIFO).

push(1)

queue: front → [1]

push(2): enqueue 2, rotate 1 to back

queue: front → [2, 1]

push(3): enqueue 3, rotate 2,1 to back

queue: front → [3, 2, 1]

pop() → popleft() → returns 3 ✓ LIFO!

queue: [2, 1]

Classic: Min Stack — O(1) getMin()

Design a stack that supports push, pop, and getMin() — all in O(1). Use a second auxiliary stack to track current minimums.

class MinStack:
  def __init__(self):
    self.stack = []
    self.mins = [] # aux min stack

  def push(self, val):
    self.stack.append(val)
    m = min(val, self.mins[-1]
           if self.mins else val)
    self.mins.append(m)

  def pop(self):
    self.stack.pop()
    self.mins.pop()

  def getMin(self):
    return self.mins[-1] # O(1)!

The mins stack always tracks the current minimum at each level — when we pop the main stack, we also pop the min stack.

push(5)

stack:[5] · mins:[5]

push(3)

stack:[5,3] · mins:[5,3]

push(7)

stack:[5,3,7] · mins:[5,3,3]

push(2)

stack:[5,3,7,2] · mins:[5,3,3,2]

getMin() → mins[-1] = 2 ✓ O(1)

No scanning needed!

Summary

When to Use Stacks vs. Queues

Use a Stack when…

  • You need to reverse a sequence
  • You need backtracking (DFS, recursion)
  • You need to match nested structures (brackets, XML)
  • You need the most recent item (undo, browser back)
  • Evaluating or converting expressions
Python stack

list with .append()/.pop() — O(1)

Use a Queue when…

  • You need to process in order of arrival (FIFO)
  • You need BFS (shortest path, level-order)
  • Task scheduling — serve requests fairly
  • Buffering — keyboard, network packets
  • Any producer-consumer communication pattern
Python queue

collections.deque with .append()/.popleft() — O(1)

Key Takeaways

  • A Stack is LIFO — last in, first out. Push and pop at the top only
  • All stack operations are O(1) — push, pop, peek
  • Stack applications: brackets, undo, call stack, DFS, expression parsing
  • Monotonic Stack solves "next greater/smaller" problems in O(n)
  • Min Stack adds O(1) getMin() using an auxiliary stack
  • A Queue is FIFO — first in, first out. Enqueue at back, dequeue at front
  • All queue operations are O(1) — enqueue, dequeue, peek
  • Queue applications: BFS, CPU scheduling, print jobs, network buffers
  • Deque generalises both — O(1) at both ends
  • Python: list for stacks · collections.deque for queues
